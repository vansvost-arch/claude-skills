# Поиск упущенного спроса — Инструкция

Анализ рекламных кампаний Яндекс.Директ для обнаружения незакрытого семантического спроса.

## Ключевые требования

- XLSX-файл с листом **«Тексты»** (стандартный экспорт из Яндекс.Директ)
- Токен `YANDEX_WORDSTAT_TOKEN` в `config/.env`
- Логин пользователя для формирования ссылок редактирования групп

## Основной процесс

1. **Парсинг** файла → таблица групп с количеством фраз
2. **Сегментация** фраз по слотам: `objects`, `actions`, `modifiers`, `additional`
3. **Сборка OR-запроса** → замер текущего спроса (X)
4. **Расширение** семантики по типу группы → замер нового спроса (Y)
5. **Отчёт** с дельтой и ссылками на редактирование

## Типы групп и стратегии

| Тип | Пример | Подход |
|-----|--------|--------|
| Транзакционная | «купить телефон» | Синонимы, действия, модификаторы |
| Брендированная | «Crosley» | Только транслит, ограниченно |
| Навигационная | «адрес магазина» | Метро, районы (только офлайн) |
| Информационная | «как выбрать» | Перефразировки, смежные задачи |

## Шаги выполнения

### Шаг 1: Парсинг XLSX

```bash
uv run --script scripts/missed_demand.py parse-xlsx путь/к/файлу.xlsx
```

Возвращает JSON со списком групп, фраз и минус-слов.

### Шаг 2: Сегментация фраз (через LLM)

Для каждой группы попроси Claude разбить фразы по слотам:

```json
{
  "objects": ["телефон", "смартфон", "мобильный"],
  "actions": ["купить", "заказать", "приобрести"],
  "modifiers": ["недорого", "б/у", "новый"],
  "additional": []
}
```

**Важные правила сегментации:**
- Расширяй только те слоты, которые **уже есть** в оригинальном запросе
- Многословный вариант, содержащий слово из однословного — избыточен
- Знаки `+`, `-`, `!`, кавычки — **не ставить вручную**, скрипт добавит сам
- При дельте >200% — проверить мусорные фразы в выдаче Wordstat

### Шаг 3: Сборка OR-запроса

```bash
echo '{"objects":["телефон","смартфон"],"actions":["купить","заказать"]}' | \
  uv run --script scripts/missed_demand.py build-query -
```

Или через merge-slots для нескольких батчей:

```bash
echo '{"phrases":[...],"batches":[...]}' | \
  uv run --script scripts/missed_demand.py merge-slots
```

### Шаг 4: Замер спроса

```bash
# Текущий спрос (X)
bash scripts/query_total.sh --phrase "исходный запрос"

# Новый спрос (Y) после расширения
bash scripts/query_total.sh --phrase "(купить|заказать) (телефон|смартфон)"
```

### Шаг 5: Отчёт

Для каждой группы:
- Текущий охват (X)
- Потенциальный охват (Y)
- Дельта: `(Y - X) / X * 100%`
- Ссылка на редактирование группы в Директе

## Ограничения

- Квота API: 10 запросов/сек, 1000/день
- `campaign_minus` часто превышает лимит в 4096 символов — применяется «best-effort»
- Скрипт автоматически обрезает запрос при превышении лимита (приоритет: objects > actions > modifiers > additional)
