## Обзор

Скилл формирует SEO-оптимизированные тексты для карточек товаров TDCompass
на основе данных спроса (Яндекс WordStat) и конкурентного анализа (SERP).

## Разрешённые инструменты

- Read, Grep, Glob — работа с файлами
- Bash — вызов скриптов WordStat и Search API
- Edit, Write — обновление файлов
- WebFetch — анализ конкурентов

## Воркфлоу

### Шаг 1: Сбор данных о спросе

```
1. Получить товар из БД (name_ru, description, category, manufacturer)
2. Сформировать поисковые запросы:
   - "{brand} {model}"
   - "{brand} {model} купить"
   - "{brand} {model} цена"
   - "{category} {brand}"
   - "{model} характеристики"
3. Запросить WordStat (скилл yandex-wordstat):
   bash scripts/top_requests.sh "{brand} {model}"
4. Получить связанные запросы и частотности
```

### Шаг 2: Анализ конкурентов в выдаче

```
1. Взять топ-5 ключей по частотности
2. Запросить SERP (скилл yandex-search-api):
   bash scripts/web_search_sync.sh "{ключ}" 225
3. Извлечь из топ-10 органики:
   - Как конкуренты называют товар (title)
   - Какие слова используют в описании (snippet)
   - Какие характеристики выделяют
```

### Шаг 3: Формирование SEO-текстов

#### Наименование товара (name_ru)
```
Правила:
- Включить главный ключ (самый частотный)
- Формат: "{Бренд} {Модель} — {тип товара} {ключевое свойство}"
- До 80 символов (для title страницы)
- Без спама, читабельно

Пример:
  Было: "Sika 268 Pro"
  Стало: "Sika 268 Pro — полиуретановый герметик для вклейки стёкол"

Пример:
  Было: "Loctite 243"
  Стало: "Loctite 243 — анаэробный фиксатор резьбы средней прочности"
```

#### Описание товара (description)
```
Правила:
- Первый абзац: 2-3 СЧ-ключа естественно
- Структура: назначение → преимущества → применение → характеристики
- Плотность ключей: 2-4% от текста
- 300-600 слов (оптимум для SEO)
- Без переспама, полезный контент
- Включить LSI-слова (семантически связанные)

Структура:
1. Вводный абзац с главным ключом и назначением
2. Преимущества (список)
3. Область применения
4. Технические характеристики (таблица)
5. Способ применения (если релевантно)
```

#### Характеристики (specifications)
```
Правила:
- Ключевые характеристики из WordStat (что люди ищут)
- Стандартные для категории (сравнимость)
- В формате: {"ключ": "значение"}

Обязательные для каждой категории:
- Герметики: тип, основа, цвет, объём, температура, время отверждения
- Клеи: тип, основа, прочность, время схватывания, материалы
- Смазки: тип, основа, температурный диапазон, NLGI, объём
```

### Шаг 4: Проверка качества

```
1. Плотность ключей 2-4%? (не больше!)
2. Текст читаемый и полезный?
3. Заголовок до 80 символов?
4. Описание 300-600 слов?
5. Нет дублирования фраз?
6. Все характеристики заполнены?
```

### Шаг 5: Запись в БД

```
Если данные спроса найдены:
  - Обновить name_ru, description, specifications в Product
  - Сохранить использованные ключи в DemandSnapshot

Если данных спроса нет (0 результатов WordStat):
  - Оставить оригинальные тексты
  - Пометить товар как "без данных спроса"
```

```
Ты — SEO-копирайтер для интернет-магазина промышленной химии TDCompass.

Товар: {name}
Бренд: {manufacturer}
Категория: {category}
Текущее описание: {description}

Ключевые слова (частотность/мес):
{keywords_with_frequencies}

Тексты конкурентов в топ-10:
{competitor_snippets}

Задача:
1. Сформируй SEO-оптимизированное наименование (до 80 символов)
   - Включи главный ключ
   - Формат: "Бренд Модель — тип товара ключевое свойство"

2. Напиши описание (300-600 слов):
   - Первый абзац с 2-3 ключевыми словами
   - Преимущества списком
   - Область применения
   - Способ применения
   - Плотность ключей 2-4%

3. Сформируй specifications (JSON):
   - Ключевые характеристики по запросам пользователей
   - Стандартные для категории

Требования:
- Естественный, полезный текст (не SEO-спам)
- Профессиональный тон (B2B аудитория)
- Конкретные факты и цифры
- Без воды и общих фраз
```
